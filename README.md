# CampusHub - 校园互助服务平台

## 📖 项目简介

这是一个连接校园内需求与服务的平台，旨在解决校园生活中信息碎片化、匹配效率低、缺乏信任机制等问题。

学生可以方便地发布需求（快递代取、学习辅导、二手交易、活动组队等），也可以浏览和接单，完成协作后获得信用分积累。


## 🛠 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端** | Vue 3 + Vite + JavaScript | 渐进式框架，快速构建 |
| **UI组件** | Element Plus | 开箱即用的组件库 |
| **状态管理** | Pinia | Vue 3 官方推荐 |
| **后端** | Node.js + Express | 轻量级 Web 框架 |
| **数据库** | MySQL | 关系型数据库 |
| **ORM** | Prisma | 类型安全的数据库工具 |
| **缓存** | Redis | 会话管理、消息队列 |
| **版本控制** | Git + GitHub | 代码管理 |
| **CI/CD** | GitHub Actions | 自动化测试与构建 |

## 🌿 分支策略

| 分支 | 说明 | 稳定性 |
|------|------|--------|
| main | 稳定生产版本，只接受来自 dev 的合并 | 非常稳定 |
| dev  | 日常开发集成分支 | 相对稳定 |
| feature | 功能开发分支，从 dev 创建，完成后合并回 dev | 不稳定 |

### 开发流程

# 1. 从 dev 创建功能分支
git checkout dev
git pull origin dev
git checkout -b feature/功能名

# 2. 开发完成后合并回 dev
git checkout dev
git merge feature/功能名
git push origin dev

# 3. 删除功能分支
git branch -d feature/功能名

## 🤝 AI 协作规范

本项目采用 AI 辅助开发，遵循以下规则：

1. **AI 是工具，人是责任主体**：所有 AI 输出必须经过人工审查
2. **标注来源**：Git commit 必须标注 [AI-assisted]或 [Human-written]
3. **保留日志**：所有与 AI 的交互 Prompt 需要记录
4. **代码审查**：AI 生成的代码必须经过至少一名团队成员 Review
5. **手写核心函数**：每人每阶段至少 2 个完全手写的核心函数

详细规范见 .github/copilot-instructions.md
