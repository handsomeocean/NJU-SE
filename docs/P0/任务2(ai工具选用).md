任务2：AI 工具选型与评估报告
一、AI 编程助手选型
（一）候选工具
1.	GitHub Copilot
2.	Cursor
（二）功能评估
GitHub Copilot 具备较强的代码补全能力，支持多语言开发，能够根据上下文生成函数、接口及基础逻辑代码。适用于日常开发场景，与主流 IDE 兼容性较好。
Cursor 是集成 AI 的编辑器，支持跨文件理解和项目级代码修改，能够对整个项目进行结构性分析与重构，适合中大型项目使用。
（三）易用性评估
GitHub Copilot 以插件形式集成到 IntelliJ 或 VS Code 中，无需更换开发环境，学习成本低。
Cursor 需要使用其独立编辑器，对已有开发环境产生替换成本，团队成员需要额外适应。
（四）成本评估
GitHub Copilot 提供学生优惠版本，费用相对可控。
Cursor 为订阅制产品，成本相对较高。
（五）团队熟悉度
团队成员已熟悉 IntelliJ 开发环境，GitHub Copilot 兼容性高。Cursor 需要重新适应新环境。
（六）最终选择
选择 GitHub Copilot 作为本项目的 AI 编程助手。
选择理由包括：兼容现有开发环境、学习成本低、功能满足课程项目需求、成本可控。
（七）未选择 Cursor 的理由
项目规模较小，暂不需要项目级重构能力；更换编辑器会增加学习成本；性价比不高。
二、AI 对话工具选型
（一）候选工具
1.	ChatGPT
2.	Claude
（二）功能评估
ChatGPT 在技术问题分析、代码逻辑解释和接口设计方面表现较强，适合生成需求分析文档与开发说明。
Claude 在长文本总结和文档整理方面表现良好，适合处理大段说明性文本。
（三）易用性评估
两者均为网页访问形式，使用门槛较低。
（四）成本评估
ChatGPT 提供免费版本，Claude 亦提供部分免费功能，整体成本差异不大。
（五）团队熟悉度
团队成员已长期使用 ChatGPT 进行技术咨询与学习，使用习惯较为成熟。
（六）最终选择
选择 ChatGPT 作为项目主要 AI 对话工具。
选择理由包括：技术解释能力更强、适合生成 API 文档、逻辑推理能力更稳定、团队熟悉度高。
（七）未选择 Claude 的理由
在复杂编程问题与调试场景中支持能力相对较弱，与团队现有使用习惯不完全匹配。
三、AI 代码审查工具选型
（一）候选工具
1.	CodeRabbit
2.	GitHub Copilot（辅助审查）
（二）功能评估
CodeRabbit 支持 Pull Request 自动分析，能够识别潜在风险与代码规范问题。
GitHub Copilot 虽不具备完整 PR 审查能力，但可在本地开发阶段提供代码优化建议。
（三）成本评估
CodeRabbit 为付费工具。GitHub Copilot 已在编程阶段使用，无需额外成本。
（四）团队熟悉度
团队未使用过 CodeRabbit，需要额外学习时间。
（五）最终选择
采用 GitHub Copilot 辅助检查 + 人工代码审查相结合的方式。
选择理由包括：课程项目规模有限、人工审查更利于理解代码逻辑、无需增加额外工具成本。
（六）未选择 CodeRabbit 的理由
成本较高，学习曲线增加，项目规模不足以体现其优势。
四、AI 测试工具选型
（一）候选工具
1.	ChatGPT（生成测试用例）
2.	Testim
（二）功能评估
ChatGPT 可根据接口说明生成单元测试示例与边界测试用例，适用于教学型项目。
Testim 为自动化测试平台，支持端到端测试与自动脚本生成，适用于企业级项目。
（三）成本评估
ChatGPT 使用成本较低。Testim 为商业化产品，费用较高。
（四）团队熟悉度
团队具备基础单元测试能力，但缺乏自动化测试平台使用经验。
（五）最终选择
选择 ChatGPT 生成测试用例，并结合人工执行测试。
选择理由包括：成本低、易于理解测试逻辑、符合课程项目规模。
（六）未选择 Testim 的理由
商业成本较高，学习成本较大，项目规模较小不需要完整自动化测试平台。
五、阶段总结
本项目在 AI 工具选型过程中，从功能适配度、易用性、成本控制及团队熟悉度四个方面进行综合评估，优先选择与现有开发环境兼容、学习成本低、能够满足课程项目需求的工具组合，在提升开发效率的同时保证技术理解与质量控制。
